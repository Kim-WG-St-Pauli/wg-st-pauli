Einmalig: Zugang einrichten

Damit dieses Werkzeug Änderungen direkt speichern kann, braucht es einmalig einen persönlichen Zugangs-Schlüssel von GitHub. Das machst du einmal, danach nie wieder.

Wie bekomme ich einen Zugangs-Schlüssel?
  • Diesen Link öffnen (am besten in einem neuen Tab): github.com/settings/personal-access-tokens/new. Falls GitHub nach dem Passwort fragt, einloggen.
  • Bei „Repository access“ die Option „Only select repositories“ wählen und wg-st-pauli auswählen.
  • Weiter unten bei „Permissions“ auf „Repository permissions“ klicken, bei „Contents“ auf „Read and write“ stellen. Mehr muss nicht angehakt werden.
  • Ganz unten auf „Generate token“ klicken. GitHub zeigt dir jetzt einmalig einen langen Code, der mit github_pat_ beginnt.
  • Diesen Code kopieren und oben ins Feld einfügen, dann auf „Verbinden“ klicken. Der Code wird nur in deinem eigenen Browser gespeichert.

Falls der Code mal nicht mehr funktioniert (z. B. weil er abgelaufen ist), einfach einen neuen nach derselben Anleitung erstellen.

Bilder & Sicherheitsnetz

Was dieses Werkzeug nicht macht

  • Ein neues Foto hochladen: diesen Link öffnen (bringt dich direkt zum Hochladen-Fenster im Ordner assets/img/) → Bild hineinziehen oder auswählen, einfacher Dateiname ohne Leerzeichen (z. B. cast-silke.jpg) → unten „Commit changes“ klicken.
  • Soll das Bild ein bestehendes Foto ersetzen: mit genau demselben Dateinamen hochladen – fertig, die Seite zeigt sofort das neue Bild, nichts weiter zu tun.
  • Ein neuer Dateiname für einen bestehenden Bewohner: Bild wie oben hochladen, dann im Werkzeug bei „Bewohner“ im Feld „Foto“ den neuen Dateinamen eintragen – das Werkzeug prüft direkt, ob es das Bild findet.
  • Einen komplett neuen Bewohner hinzufügen, der noch gar nicht existiert: das ist ein größerer Eingriff (neue Person im Datenmodell), das kann das Werkzeug noch nicht. Dafür kurz den Helfer ansprechen – am einfachsten mit Name der Person, ein bis zwei Sätzen zur Figur und dem gewünschten Foto.

Nichts ist endgültig kaputtbar. Jede Veröffentlichung ist ein Punkt in der Datei-Historie bei GitHub, nichts wird je gelöscht. Geht doch mal was schief: genaue Anleitung, wie du eine Änderung rückgängig machst ↗. Außerdem prüft die Seite jede Änderung automatisch, bevor sie live geht: Ist etwas unvollständig, wird einfach nicht neu veröffentlicht und der letzte gute Stand bleibt online.

Damit Videos überall laufen: Vimeo-Videos können auf bestimmte Domains beschränkt sein. In den Vimeo-Einstellungen jedes Videos unter Privatsphäre → Wo darf eingebettet werden die Domain(s) ergänzen, unter denen die Seite erreichbar ist.
